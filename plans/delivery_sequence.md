# Delivery Sequence

## Phase 1
- finalize repository framing
- define prototype input contracts
- define normalized asset and lifecycle policy contracts

## Phase 2
- choose first implementation mode:
  API-first, dashboard-first, or report-first
- choose language and runtime
- define persistence boundary

## Phase 3
- implement inventory collection
- normalize assets
- ingest lifecycle policy data

## Phase 4
- implement deterministic lifecycle evaluation
- implement scoring and prioritization
- generate per-asset assessments

## Phase 5
- generate AI briefs and portfolio summaries
- attach evidence and confidence
- define review and approval workflow

## Phase 6
- add workflow integrations
- add non-production rehearsal support
- add operating controls and auditability
