# Assistant Operating Contract

## Purpose
Define how AI should operate inside this product.

## Contract
- authoritative lifecycle and policy facts must come from deterministic, refreshable data
- AI may summarize, explain, compare, rank narrative emphasis, and draft next-step guidance
- AI outputs must retain evidence references and confidence indicators
- production execution requires explicit human approval
- missing evidence must be surfaced, not filled with speculation

## Prompting boundary
Prompts should be grounded in normalized asset data, lifecycle policy records, deterministic assessment output, and source references.

## Audit boundary
The system should preserve:
- assessment inputs
- rule outputs
- prompt context
- generated brief content
- model identity
- generation timestamp
